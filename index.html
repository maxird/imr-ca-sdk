Welcome

Welcome to the new California IMR Document Exchange Portal Sandbox. The goal of the sandbox is to offer users an opportunity to try out the various features of the site in a test environment. Because the sandbox is hosted publically on Amazon’s web cloud service, it is essential that real medical records are never uploaded to the site.

Accessing the Portal

In order to log into the portal, users must first be assigned a user name and password. Within each Claims Administrator and Utilization Review organization participating in the pilot, a designated administrator has been selected to manage the group’s users. The portal supports 2 different mechanisms to provision new users. The first method requires the administrator to use to the available KeyCloak console to register a user. The second method involves completing a user enrollment spreadsheet and submitting it to Maximus for processing. Once a user is registered, an email notification is issued by the portal to the user further log-in instructions.

Available Web Services

Several key portal functions are exposed as REST style services. These include:

  • Retrieving NOARFI Notices by date range. The service also includes an ability to filter out cases for which at least one document was already received through the portal from the requestor.
  • Uploading documents to a case. Files can be either be uploaded with a file name that is in conformance with existing MoveIt naming standard for automatic case mapping or an additional case number can be supplied with files containing a non-standard file name.
  • Retrieving a list of all documents for a specified case. Document list details include a unique document id, the creation date, the last updated date, source of the document, and type of document.
  • Requesting a specific document using its unique document id
  • Obtaining a list of active cases. Case list details include the case’s id, current case status, case status date, and the date the case was last updated.
An available SDK offers detailed documentation for each available web service along with working Java and JavaScript sample clients.
